## Этап 1: Создать универсальный `<QuickLeadModal />`

**Файл:** `apps/web/src/components/forms/quick-lead-modal.tsx` (новый)

### Интерфейс

```typescript
interface QuickLeadModalProps {
  open: boolean;
  onOpenChange: (open: boolean) => void;
  title?: string;                    // default: "Оставить заявку"
  description?: string;              // default: "Перезвоним в течение 5 минут"
  source: string;                    // для /api/contacts + analytics (max 100 симв)
  context?: Record<string, string>;  // скрытые данные (zone, fleet, package, price...)
  submitLabel?: string;              // default: "Получить расчёт"
  withTurnstile?: boolean;           // default: false (включать для публичных форм)
}
```

### Поведение

1. **Responsive render:**
   - На мобильном (<768px) — `<Sheet side="bottom">` (как StickyCTA, проверено работает)
   - На десктопе — `<Dialog>`
   - Детектить через `useMediaQuery` или существующий хелпер

2. **Поля:**
   - Имя (text, required)
   - Телефон (tel, required, автоформатирование) — можно взять validator из `InlineLeadForm`
   - Согласие на обработку (чекбокс, required, с ссылкой на `/privacy`)

3. **UTM capture:**
   ```typescript
   function readUtmFromCookies() {
     if (typeof document === "undefined") return {};
     const cookies = Object.fromEntries(
       document.cookie.split("; ").map(c => c.split("=").map(decodeURIComponent))
     );
     return {
       utm_source: cookies.utm_source,
       utm_medium: cookies.utm_medium,
       utm_campaign: cookies.utm_campaign,
       utm_content: cookies.utm_content,
       utm_term: cookies.utm_term,
     };
   }
   ```

4. **Submit:**
   - `POST /api/contacts`:
     ```json
     {
       "name": "...",
       "phone": "+7...",
       "source": "<source>",  // max 100
       "context": { ...utm, ...props.context }  // новое поле Zod-схемы
     }
     ```
   - Если 429 (rate-limit) → `toast.error("Слишком много запросов. Подождите минуту.")`
   - Если 200 → показать зелёный чек, текст «Перезвоним за 5 минут», авто-закрытие через 3 сек
   - Loading state на кнопке (disabled + спиннер)
   - Двойной submit заблокирован через `loading`

5. **Analytics events:**
   - `analytics.leadModalOpened(source)` — при открытии
   - `analytics.callbackRequested()` — при успехе
   - `analytics.leadModalClosed(source, submitted: boolean)` — при закрытии (для воронки abandoned)

6. **A11y:**
   - Автофокус на поле «Имя» при открытии
   - `aria-describedby` на description
   - Focus trap — через shadcn Dialog/Sheet из коробки

7. **Honeypot anti-bot:**
   - Скрытое поле `website` (стандартный honeypot). Если заполнено — silent fail (200 OK, но заявка не создаётся).
   - Без Turnstile на MVP (он требует site key от пользователя).

**Размер компонента:** ~200 строк.

## Этап 10: Обновить `src/lib/analytics/events.ts`

Добавить новые события:
```typescript
export const analytics = {
  // ...existing
  leadModalOpened: (source: string) => trackEvent("lead_modal_opened", { source }),
  leadModalSubmitted: (source: string) => trackEvent("lead_modal_submitted", { source }),
  leadModalClosed: (source: string, submitted: boolean) =>
    trackEvent("lead_modal_closed", { source, submitted: submitted ? 1 : 0 }),
};
```

Это позволит настроить воронку в Метрике: opened → submitted → success. **Abandonment rate** покажет где форма теряет пользователей.
