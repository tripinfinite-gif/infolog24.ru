### Обнаруженные проблемы

**Критические:**
- ⚠️ **Дублирование ЛК-планов.** `личный-кабинет.md` и `личный-кабинет-фронтенд.md` описывают одно и то же. **Решение:** использовать `личный-кабинет-фронтенд.md` как основной, второй — архивировать.
- ⚠️ **Нет плана админ-панели менеджеров.** В `автоматизация-и-уведомления.md` есть фаза 7 "Панель менеджера", но нет отдельного плана. Менеджерам нужен полноценный интерфейс для обработки 600-800 заявок/мес.

**Средние:**
- Перекрытие фаз SEO между `редизайн-сайта` (фаза 7) и `seo-аналитика-интеграции` — **Решение:** SEO-план считать основным, из редизайна убрать.
- В `бэкенд-и-api` и `автоматизация` описана одна и та же БД-схема — нужен единый source of truth.
**Некритические:**
- Бюджет на AI-чатбот ($13-97/мес) — заложить запас при росте трафика.
- Calltouch (трекинг звонков) стоит отдельных денег — уточнить бюджет.

| Элемент | Описание | Где добавить |

### Супер-фичи (конкурентное преимущество)

| Фича | Описание | Влияние на бизнес |
|------|----------|-------------------|
| **Онлайн-калькулятор стоимости** | Интерактивный калькулятор: тип пропуска × зона × срок × кол-во ТС → цена. Сразу кнопка "Оформить" | ↑ Конверсия на 15-25%. Главный лид-магнит |
| **Проверка статуса пропуска** | Публичная страница: ввёл номер заявки → видишь статус. Снижает нагрузку на менеджеров | ↓ Звонки "где мой пропуск?" на 30-40% |
| **Партнёрский портал** | ЛК для партнёров: реферальная ссылка, статистика приведённых клиентов, вывод вознаграждения | Новый канал привлечения, 6 партнёров уже есть |
| **Автоматическое напоминание о продлении** | За 30/14/7 дней до истечения пропуска — SMS + Telegram + email | ↑ Retention с 70% до 85%+ |
| **Шаблоны документов для скачивания** | Доверенность, заявление, чеклист документов — PDF-генерация | SEO-трафик + доверие + снижение ошибок |
| **Виджет "Перезвоните мне"** | Callback-форма с выбором времени + интеграция с менеджерами через Telegram | ↑ Конверсия из посетителя в лид |
| **Дашборд руководителя** | Метрики: заявки/день, конверсия, средний чек, загрузка менеджеров, воронка | Управленческие решения на данных |
| **Email-цепочки (drip campaigns)** | Автоматические письма: после заявки, после оплаты, через 30 дней, перед продлением | ↑ LTV, ↓ отток |
| **Мультиязычность (RU/EN)** | Минимально: англ. версия главной + ЛК для иностранных перевозчиков | Расширение аудитории |
| **PWA-режим** | Иконка на домашний экран телефона, push-уведомления, офлайн-кэш | Удержание мобильных пользователей |

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| YooKassa долгое подключение | Средняя | Среднее | Начать процесс KYC заранее (фаза 0) |
| Перегруз инфры при запуске | Низкая | Высокое | Нагрузочные тесты (фаза 4), CDN через Cloudflare |
| Bitrix24 API нестабильно | Средняя | Среднее | Очередь с retry, fallback на ручной режим |
| Контент не готов вовремя | Высокая | Высокое | Начать фазу 1C первой, параллельно с инфрой |
| 152-ФЗ: данные за пределами РФ | Низкая | Критическое | Selectel (Москва), без зарубежных SaaS для данных |
| Сложность BullMQ + Redis | Средняя | Среднее | Начать с простых cron-задач, BullMQ — по мере роста |
| Перегрузка первой версии фичами | Высокая | Среднее | MVP-подход: Фазы 0-2 = запуск, Фазы 3-6 = развитие |

## Итого

**13 фаз, 140+ задач, 6+ параллельных потоков.**

Проект можно реализовать за **12-14 недель** при наличии 3-4 агентов, работающих параллельно. Максимальная параллельность — на фазах 1 (3 агента) и 2-3 (4 агента).

### Что добавлено сверх существующих планов

**Новые модули:**
- Админ-панель менеджеров (Фаза 2D) — 15 задач, без неё бизнес не работает
- Партнёрский портал (Фаза 3D) — канал уже есть (6 клиентов)

**Супер-фичи:**
- Онлайн-калькулятор стоимости — главный конверсионный элемент (+15-25%)
- Проверка статуса пропуска — снижение звонков на 30-40%
- Автонапоминание о продлении — retention с 70% до 85%+
- Система промокодов и скидок — гибкость маркетинга
- Дашборд руководителя — управление на данных
- Email-цепочки (drip campaigns) — увеличение LTV
- PWA-режим — удержание мобильных

**Технические элементы:**
- Серверное логирование (pino)
- Error boundary + fallback UI
- Redis кэширование + BullMQ очередь задач
- Health check эндпоинт
- Env-валидация (`@t3-oss/env-nextjs`)
- Версионирование API (`/api/v1/*`)
- Cookie consent баннер
- Status page

**Бизнес-процессы:**
- Внутренний чат менеджеров (комментарии к заявкам)
- Шаблоны ответов для менеджеров
- Система тегов для заявок
- Экспорт данных (CSV/Excel)
- Автоматический расчёт сроков готовности
- Автоформирование актов выполненных работ
