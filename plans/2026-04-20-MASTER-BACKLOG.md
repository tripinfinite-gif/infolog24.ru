## 🔥 P0 — БЛОКЕРЫ (юридический риск, блокировка сайта)

### P0-1. Переформулировать «оформляем пропуска» → «помощь с документами»
**Проблема**: мы НЕ имеем права оформлять пропуска. Сайты с такой формулировкой блокируются.

**Действие**: глобальный поиск+замена по всему коду и контенту:

| ❌ Убрать | ✅ Заменить на |
|---|---|
| «Оформим пропуск» | «Поможем с документами для получения пропуска» |
| «Оформление пропуска» | «Сопровождение оформления пропуска» |
| «Оформляем пропуска» | «Помогаем с получением пропусков» |
| «Получим пропуск» | «Поможем получить пропуск» |
| «Выдадим пропуск» | «Поможем в получении» |

**Локации**: заголовки страниц, H1, CTA-кнопки, meta-title, meta-description, OG, sitemap, breadcrumbs, FAQ, договор, оферта, политика, alt-теги, ИИ-ассистент (промпт + datasets), датасеты диалогов.

**Команды поиска**:
```
grep -r "оформ.*пропуск" apps/ packages/
grep -r "получим пропуск\|выдадим пропуск\|выпустим пропуск" apps/ packages/
grep -ri "офорМление пропуск" apps/ packages/
```

### P1-6. Ночные пропуска — НЕ отменены
**Правка сотрудника-2 неточна**: он сказал «ночные заменить на дневные (они же круглосуточные)».

**Корректно по нормативке**:
- В 379-ПП **ночной и дневной пропуска — разные документы**.
- Для ТС > 3,5 т по МКАД с 01.01.2022 нужен **круглосуточный** (как единый документ или пара «дневной+ночной»).
- «Дневной = круглосуточный» — маркетинговая фраза посредников, не норма.

**Формулировка для сайта**:
✅ «Помогаем с получением пропусков — дневной, ночной, круглосуточный».

**Локация**: калькулятор (тултип «Цены от. Экокласс, ночные пропуски, доп. опции…»), страницы зон, FAQ.

**Действие**:
- Разделить карточку на две: «Для экспедиторов (срочно — до 30.04.2026)» + «Для перевозчиков (с 01.03.2027)»

## 🔴 P3 — UI-БАГИ (срочно, ломают UX)

### P3-1. Битая кодировка (�� replacement character)
**Найдено**:
- «требовани��м реестра» → требованиям
- «ни��аких скрытых доплат» → никаких
- «дл�� регистрации» → для
- «Какие документы нужны дл�� регистрации?» → для

**Действие**:
```bash
grep -rn $'\uFFFD' apps/ packages/     # replacement char
grep -rn "��" apps/ packages/          # визуальный артефакт
```
+ проверить charset БД и файлов, meta `<charset>`, Content-Type headers.
+ прогнать весь сайт визуально (может быть больше мест).
