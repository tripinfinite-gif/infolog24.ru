## Результат исследования (апрель 2026)

### Ключевые решения на основе анализа

| Задача | Выбор | Почему |
|---|---|---|
| Микроразметка | **JSON-LD** (не microdata, не RDFa) | Google и Яндекс рекомендуют JSON-LD. Проще поддерживать, не засоряет HTML, легко генерировать динамически в Next.js |
| Защита форм | **Cloudflare Turnstile** (не reCAPTCHA) | Бесплатный, GDPR-совместимый, невидимый для пользователя, не блокируется в РФ (в отличие от reCAPTCHA, у которой бывают проблемы с доступностью Google-сервисов) |
| Аналитика | **Яндекс.Метрика** (основная) + GA4 (вторичная) | Метрика — стандарт для Рунета, Вебвизор бесплатный, лучше считает российский трафик. GA4 — для полноты данных |
| Коллтрекинг | **Calltouch** | Лидер рынка РФ, стабильная интеграция с Битрикс24, подменные номера, сквозная аналитика, API для Next.js |
| Онлайн-чат | **JivoSite** | Наибольшая база в РФ, интеграция с Битрикс24 / WhatsApp / Telegram, бесплатный тариф для старта, хорошая мобильная версия |
| Sitemap / robots | **Next.js built-in** (sitemap.ts, robots.ts) | Нативная поддержка в App Router, автогенерация, TypeScript-типизация |
| Meta-теги | **Next.js Metadata API** (generateMetadata) | Встроенная поддержка, серверный рендер, автоматический OG |
| 301-редиректы | **next.config.js** (статические) + **middleware** (динамические) | next.config.js для простых 1:1 маппингов, middleware для паттернов с regex и условной логикой |

## Фаза 1. Микроразметка Schema.org

> Цель: rich snippets в Google и Яндекс, повышенный CTR в выдаче.

### 1.1 Архитектура разметки

Каждый тип разметки реализуется как React-компонент, выводящий `<script type="application/ld+json">` в `<head>`.

```
src/components/seo/
├── JsonLd.tsx              ← базовый компонент обёртка
├── LocalBusinessJsonLd.tsx ← главная, контакты
├── ServiceJsonLd.tsx       ← страницы услуг
├── FaqJsonLd.tsx           ← FAQ-секции
├── ReviewJsonLd.tsx        ← отзывы
├── ArticleJsonLd.tsx       ← блог
├── BreadcrumbJsonLd.tsx    ← хлебные крошки
└── PriceJsonLd.tsx         ← страница цен
```

## Фаза 10. Core Web Vitals

> Цель: PageSpeed Insights 90+ (мобайл и десктоп).

### 10.1 Метрики (актуальные на 2026)

| Метрика | Хорошо | Нужно улучшить | Плохо |
|---|---|---|---|
| **LCP** (Largest Contentful Paint) | ≤ 2.5s | ≤ 4.0s | > 4.0s |
| **INP** (Interaction to Next Paint) | ≤ 200ms | ≤ 500ms | > 500ms |
| **CLS** (Cumulative Layout Shift) | ≤ 0.1 | ≤ 0.25 | > 0.25 |

**INP** заменил FID в марте 2024. Измеряет задержку ВСЕХ взаимодействий (клики, тапы, нажатия клавиш), а не только первого.

### 10.2 Оптимизация

- [ ] **10.2.1** **LCP:** 
  - Hero-изображения: `<Image priority>` (Next.js preload)
  - Шрифты: `next/font` с `display: swap` + preload
  - Минимизировать CSS в `<head>` (Tailwind purge)
  - SSR / SSG для критического контента
- [ ] **10.2.2** **INP:**
  - Скрипты аналитики — `strategy: "afterInteractive"` или `"lazyOnload"`
  - JivoSite — `strategy: "lazyOnload"` (не блокирует main thread)
  - Calltouch — `strategy: "afterInteractive"` (нужен для подмены номеров)
  - Тяжёлые обработчики — `startTransition` / `useTransition`
  - Калькулятор — debounce пересчёта (150ms)
- [ ] **10.2.3** **CLS:**
  - Все изображения: `width` + `height` (или `fill` с контейнером)
  - Шрифты: `size-adjust` через `next/font` (автоматически)
  - Placeholder для lazy-loaded контента (skeleton)
  - JivoSite виджет: зарезервировать место в DOM

### 10.3 Мониторинг

- [ ] **10.3.1** PageSpeed Insights — проверять после каждого деплоя
- [ ] **10.3.2** Google Search Console → Core Web Vitals отчёт (реальные данные)
- [ ] **10.3.3** `web-vitals` npm пакет — отправка метрик в Яндекс.Метрику:
  ```tsx
  import { onLCP, onINP, onCLS } from 'web-vitals'
  
  onLCP((metric) => reachGoal('web_vital_lcp', { value: metric.value }))
  onINP((metric) => reachGoal('web_vital_inp', { value: metric.value }))
  onCLS((metric) => reachGoal('web_vital_cls', { value: metric.value }))
  ```
