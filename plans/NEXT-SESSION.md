## 1. Контекст: что за проект

**Инфологистик-24** — сервис оформления пропусков для грузового транспорта в Москву (МКАД, ТТК, Садовое кольцо). Компания работает с 2016 года, 30–50 сотрудников, ~600–800 заявок/мес, выручка ~17 млн/мес.

Мы строим **полную цифровую платформу**: маркетинговый сайт + личный кабинет клиента + админ-панель менеджеров + автоматизация + AI-агент **«ИнфоПилот»** (см. ниже).

**Стратегическое переосмысление (сессия 4, 2026-04-11):** ЛК больше не «форма заявки на пропуск», а **операционная система перевозчика** с реестрами всех данных парка (R1–R20), тирами клиентов (T1–T5) и встроенным голосовым AI-агентом, который читает данные клиента и помогает с пропусками, штрафами, документами, эвакуациями и юр.вопросами. См. план [`2026-04-11-личный-кабинет-v2-реестры-и-ai-агент.md`](2026-04-11-личный-кабинет-v2-реестры-и-ai-агент.md).

## 11. Рекомендация для следующей сессии

**Пользователь только что переосмыслил весь личный кабинет.** Логичный следующий шаг — **не начинать сразу кодить ЛК v2**, а:

1. **Доделать инфраструктуру** (Вариант A) — без работающих миграций и БД невозможно стартовать ни один реестр.
2. **Параллельно** — попросить пользователя инициировать BizDev-задачи: договор с Yandex Cloud (долгий процесс), запрос доступа к MAX bot API (закрытая бета, может занять время), переговоры с РНИС-операторами / Платоном / ЭДО / партнёрской сетью ИнфоПилота.
