# AI-консультант: интеграция KB в рабочее приложение

**Дата:** 2026-04-16
**Статус:** активный
**Размер:** L
**Предшествует:** `2026-04-15-ai-консультант-knowledge-base.md` (пайплайн закрыт)

## Контекст и стратегия

В `apps/web` уже есть полноценный чат с AI-помощником:
- `src/content/knowledge-base.ts` — 140 items (short/detail/aliases/tags, 18 категорий)
- `src/lib/chat/system-prompt.ts` — развёрнутый промпт с правилами, продуктовым контекстом, эскалацией
- `src/lib/chat/hybrid-search.ts` — keyword + pgvector RRF (1536 dim, OpenAI embeddings)
- `src/lib/chat/tools.ts` — Vercel AI SDK tools (searchKnowledge, getKnowledgeDetail, cabinet tools, escalation)
- Таблица `knowledge_chunks` в drizzle миграции 0003

**Стратегия:** не переписывать, а обогащать. Наша KB из транскриптов даёт:
1. Живые alt-формулировки клиентов (улучшают keyword-поиск)
2. Реальные вопросы, которых нет в curated-базе (AnyDesk, жаргон, edge cases)
3. Продающие техники из playbook (Empathy-first, ВЭРВО, 5 механик)
4. Горящие темы 2025-2026 из внешних источников (РНИС волна, 7500 ₽, карантин)

## Фазы

### Фаза I.1 — Playbook-модуль для промпта
Выгрузить техники из `analysis/ai-consultant-kb/playbook/*.md` в `apps/web/src/lib/chat/playbook.ts` как exported константы (RESPONSE_PATTERNS, OBJECTION_FORMULAS, HOT_TOPICS_2026, CUSTOMER_JARGON).

### Фаза I.2 — Enriched knowledge items
Добавить новый файл `src/content/knowledge-base-transcripts.ts` с ~15-20 действительно новыми items, которых нет в текущей базе (AnyDesk удалённое подключение, РНИС 30-дневный карантин, грузовой каркас vs пропуск, клиентский жаргон). Подключить в главный knowledgeBase массив merge-сплитом.

### Фаза I.3 — Enrich existing aliases
Скрипт `scripts/enrich-kb-aliases.ts` — для каждой совпадающей темы добавить alt_formulations из транскриптов в поле `aliases` у существующих items. Prevent duplicates.

### Фаза I.4 — System prompt enhancements
Обновить `src/lib/chat/system-prompt.ts`:
- Секция "ГОРЯЩИЕ ТЕМЫ 2025-2026" (РНИС массовые аннуляции, штраф 7500/10000, скидка 25%, 30-дневный карантин, СВХ ужесточение)
- Секция "ПРОДАЮЩИЕ ТЕХНИКИ" (empathy-first, ВЭРВО, 5 механик топ-менеджеров, живой язык перевозчика, клиентский жаргон)
- Секция "НА ЧТО ОБРАЩАТЬ ОСОБОЕ ВНИМАНИЕ" (конкуренты, «кидалы», 3 знака недоверия)

### Фаза I.5 — Ingest transcript KB в pgvector
Скрипт `scripts/seed-transcript-chunks.ts`:
- Читает `analysis/ai-consultant-kb/index/answers.jsonl`
- Генерирует OpenAI embeddings (1536 dim, `text-embedding-3-small`)
- Upsert в `knowledge_chunks` с префиксом `source_id = 'transcript:<id>'`
- Не трогает существующие записи curated base

### Фаза I.6 — Hybrid search включает transcript source
Обновить `src/lib/chat/hybrid-search.ts` чтобы корректно обрабатывать и curated, и transcript источники в одной выдаче с дедупликацией по теме.

### Фаза I.7 — Тесты и верификация
- Unit-тест: playbook константы экспортируются и непустые.
- Unit-тест: knowledge-base-transcripts items валидны (все поля).
- Интеграционный smoke: buildSystemPrompt содержит новые секции.
- `pnpm build` проходит.
