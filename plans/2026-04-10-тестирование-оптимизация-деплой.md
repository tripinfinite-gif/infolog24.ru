### 2.2. Оптимизация LCP (Largest Contentful Paint < 2.5s)

LCP — время отрисовки самого крупного видимого элемента (обычно hero-изображение или заголовок).

- [ ] **Изображения:**
  - Все изображения в формате **WebP/AVIF** через `next/image`
  - Hero-изображение: `priority` + `fetchPriority="high"` + `sizes`
  - Preload hero-изображения в `<head>`:
    ```tsx
    <link rel="preload" as="image" href="/hero.webp" />
    ```
  - Lazy loading для всех изображений ниже fold: `loading="lazy"`

- [ ] **Шрифты:**
  - Использовать `next/font` (self-hosted, без запроса к Google Fonts)
  - `font-display: swap` (текст виден сразу, шрифт подгружается)
  - Preload основного шрифта:
    ```tsx
    <link rel="preload" as="font" type="font/woff2" href="/fonts/inter.woff2" crossorigin />
    ```

- [ ] **Серверный рендеринг:**
  - Главная, услуги, цены, FAQ — **статическая генерация** (SSG) через `generateStaticParams`
  - Блог — ISR (Incremental Static Regeneration) с `revalidate: 3600`
  - ЛК — серверные компоненты (RSC), не клиентский рендеринг

- [ ] **Критический CSS:**
  - Tailwind CSS с `content` настроен правильно — неиспользуемые классы удаляются при сборке
  - Проверить размер CSS-бандла: `pnpm build && ls -la .next/static/css/`

### 2.3. Оптимизация CLS (Cumulative Layout Shift < 0.1)

CLS — визуальная стабильность, сдвиги элементов при загрузке.

- [ ] **Изображения и видео:**
  - Все `<Image>` имеют `width` и `height` (или `fill` с контейнером)
  - Нет изображений без заданных размеров
  - Skeleton-заглушки для динамически загружаемого контента

- [ ] **Шрифты:**
  - `font-display: swap` + `size-adjust` в `next/font` (предотвращает FOUT-сдвиг)
  - Fallback-шрифт подобран по метрикам к основному

- [ ] **Динамический контент:**
  - Виджет чата загружается в фиксированном контейнере (не сдвигает страницу)
  - Баннеры/уведомления имеют зарезервированное пространство
  - Lazy-loaded секции имеют `min-height`

- [ ] **Рекламные и внешние блоки:**
  - Яндекс.Метрика, GA4 — загружать асинхронно (`async` / `defer`)
  - Виджеты (чат, коллтрекинг) — загружать после `onload` или по `requestIdleCallback`

### 2.4. Оптимизация INP (Interaction to Next Paint < 200ms)

INP — отзывчивость на действия пользователя.

- [ ] **JavaScript-бандл:**
  - Проверить размер бандла: `pnpm --filter web build && npx @next/bundle-analyzer`
  - Тяжёлые библиотеки — dynamic import: `next/dynamic` с `{ ssr: false }`
  - Калькулятор стоимости — клиентский компонент, но лёгкий (без тяжёлых зависимостей)

- [ ] **Обработчики событий:**
  - Формы — не блокируют UI при отправке (показать spinner/disabled)
  - Фильтры и поиск — debounce 300ms
  - Scroll-обработчики — passive: true, throttle

- [ ] **Third-party скрипты:**
  - Загрузка внешних скриптов через `next/script` с `strategy="lazyOnload"`:
    ```tsx
    <Script src="https://mc.yandex.ru/metrika/tag.js" strategy="lazyOnload" />
    ```

### 4.2. Сценарии нагрузочного тестирования

```javascript
// tests/load/api-load.js
import http from 'k6/http';
import { check, sleep } from 'k6';

export const options = {
  stages: [
    { duration: '1m', target: 20 },   // Ramp-up: 0 → 20 пользователей
    { duration: '3m', target: 50 },   // Нагрузка: 50 пользователей
    { duration: '2m', target: 100 },  // Пиковая: 100 пользователей
    { duration: '1m', target: 0 },    // Ramp-down
  ],
  thresholds: {
    http_req_duration: ['p(95)<500'],  // 95% запросов < 500ms
    http_req_failed: ['rate<0.01'],    // < 1% ошибок
  },
};

const BASE_URL = 'https://staging.infolog24.ru';

export default function () {
  // Сценарий 1: Просмотр публичных страниц
  const pages = ['/', '/services/mkad', '/pricing', '/faq', '/about'];
  const page = pages[Math.floor(Math.random() * pages.length)];
  
  const res = http.get(`${BASE_URL}${page}`);
  check(res, {
    'status 200': (r) => r.status === 200,
    'response time < 500ms': (r) => r.timings.duration < 500,
  });

  sleep(1);

  // Сценарий 2: Отправка формы заявки
  const formRes = http.post(`${BASE_URL}/api/orders`, JSON.stringify({
    name: 'Нагрузочный Тест',
    phone: '+79001234567',
    zone: 'mkad',
    permit_type: 'annual',
  }), {
    headers: { 'Content-Type': 'application/json' },
  });
  check(formRes, {
    'form submit 200/201': (r) => [200, 201].includes(r.status),
    'form response < 1s': (r) => r.timings.duration < 1000,
  });

  sleep(2);
}
```

### 4.3. Сценарий нагрузки на API ЛК

```javascript
// tests/load/dashboard-load.js
import http from 'k6/http';
import { check, sleep } from 'k6';

export const options = {
  vus: 30,          // 30 одновременных пользователей ЛК
  duration: '5m',
  thresholds: {
    http_req_duration: ['p(95)<1000'],  // API ЛК — 95% < 1s
    http_req_failed: ['rate<0.02'],
  },
};

const BASE_URL = 'https://staging.infolog24.ru';

export default function () {
  // Авторизация
  const loginRes = http.post(`${BASE_URL}/api/auth/sign-in`, JSON.stringify({
    email: `loadtest${__VU}@test.com`,
    password: 'testpassword',
  }), {
    headers: { 'Content-Type': 'application/json' },
  });

  const token = loginRes.cookies['session_token']?.[0]?.value;

  // Запросы ЛК
  const headers = {
    'Content-Type': 'application/json',
    'Cookie': `session_token=${token}`,
  };

  // Список заявок
  check(http.get(`${BASE_URL}/api/orders`, { headers }), {
    'orders list 200': (r) => r.status === 200,
  });
  sleep(1);

  // Список ТС
  check(http.get(`${BASE_URL}/api/vehicles`, { headers }), {
    'vehicles list 200': (r) => r.status === 200,
  });
  sleep(1);

  // Уведомления
  check(http.get(`${BASE_URL}/api/notifications`, { headers }), {
    'notifications 200': (r) => r.status === 200,
  });
  sleep(2);
}
```

### 4.6. Оптимизация по результатам

Если метрики не проходят:

| Проблема | Диагностика | Решение |
|----------|-------------|---------|
| Медленные запросы к БД | `EXPLAIN ANALYZE` в PostgreSQL | Добавить индексы, оптимизировать запросы Drizzle |
| Высокий CPU | `htop` на VPS | Кэширование Redis, увеличить VPS |
| Медленные страницы | Lighthouse | SSG вместо SSR, CDN для статики |
| Утечки памяти | Node.js `--inspect`, memory profiling | Исправить утечки, рестарт-политика в Coolify |

## Фаза 5. Тестирование безопасности

> Минимальный security-чек перед выходом в production.

### 5.1. Заголовки безопасности

- [ ] Проверить HTTP-заголовки:
  ```bash
  curl -I https://staging.infolog24.ru
  ```
- [ ] Убедиться в наличии:
  - `Strict-Transport-Security: max-age=31536000; includeSubDomains`
  - `X-Content-Type-Options: nosniff`
  - `X-Frame-Options: DENY` (или `SAMEORIGIN`)
  - `Referrer-Policy: strict-origin-when-cross-origin`
  - `Content-Security-Policy` (хотя бы базовый)

### 5.2. Проверка форм

- [ ] reCAPTCHA / Turnstile работает на всех формах
- [ ] Rate limiting на API-эндпоинтах (`/api/orders`, `/api/auth/sign-in`)
- [ ] XSS — ввод `<script>alert(1)</script>` в поля форм не выполняется
- [ ] CSRF — токены присутствуют (Better Auth обеспечивает)
- [ ] SQL Injection — Drizzle ORM параметризует запросы (проверить raw-запросы)

### 5.3. Аутентификация

- [ ] Пароли хешируются (Better Auth — bcrypt/argon2)
- [ ] Сессии истекают (проверить TTL)
- [ ] Недоступные маршруты возвращают 401/403, не данные
- [ ] API-эндпоинты ЛК недоступны без авторизации

### 5.4. Файлы и загрузки

- [ ] Загрузка файлов — проверка MIME-type на сервере (не только расширение)
- [ ] Максимальный размер файла ограничен (10 MB)
- [ ] Файлы в S3 недоступны по прямому URL без подписанного токена
- [ ] `.env`, `node_modules`, `.git` — недоступны снаружи

### 8.1. GitHub Actions для тестов

```yaml
# .github/workflows/ci.yml
name: CI
on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  lint-and-type-check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 24
          cache: pnpm
      - run: pnpm install --frozen-lockfile
      - run: pnpm lint
      - run: pnpm type-check

  e2e-tests:
    runs-on: ubuntu-latest
    needs: lint-and-type-check
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 24
          cache: pnpm
      - run: pnpm install --frozen-lockfile
      - run: pnpm exec playwright install --with-deps
      - run: pnpm build
      - run: pnpm exec playwright test
      - uses: actions/upload-artifact@v4
        if: failure()
        with:
          name: playwright-report
          path: playwright-report/
```
