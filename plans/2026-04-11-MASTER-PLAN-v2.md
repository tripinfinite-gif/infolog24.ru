## 1. Зачем v2 — что изменилось

Первый мастер-план (2026-04-10) был создан вместе с кодовой базой за одну сессию. Все 140+ задач помечены как `[x]`. Однако **ревизия кода показала**, что реальный статус — **каркас на моках**:

| Что в v1 значилось как `[x]` | Реальный статус |
|---|---|
| Миграции БД применены | Схема написана, но папка `drizzle/` пустая — **миграции не запускались** |
| ЛК клиента работает | UI-страницы с **mock-данными**, не подключены к DAL |
| Админ-панель работает | UI-страницы с **mock-данными**, не подключены к DAL |
| Партнёрский портал работает | Скаффолд, не подключён |
| Email/SMS уведомления работают | Модули определены, **отправка не подключена** |
| S3 загрузка файлов | Endpoint есть, **интеграция не завершена** |
| BullMQ очереди работают | Пакет установлен, **очереди не определены** |
| Маркетинговые страницы готовы | Контент написан, но **часть страниц удалена** в текущих изменениях |
| SEO/аналитика подключена | **Не настроено** |
| Тесты написаны | **Ни одного теста** |
| Деплой настроен | **VPS не заказан, CI/CD нет** |

### Что реально работает

- Монорепо (Turborepo + pnpm) — настроено
- Схема БД (Drizzle ORM, 15+ таблиц) — написана
- Better Auth (email+пароль, 2FA, роли) — сконфигурирован
- shadcn/ui компоненты — установлены и кастомизированы
- API routes (20+ endpoints) — написаны, используют DAL
- DAL (Data Access Layer) — реализован для основных сущностей
- Контент (тексты, тарифы, FAQ, блог) — подготовлен
- AI-чатбот (Vercel AI SDK + OpenAI) — каркас с rate limiting
- Telegram-бот (grammY) — каркас с webhook
- Автоматизация (state machine, document checker) — каркас
- ESLint flat config (v9) — настроен, lint + tsc проходят чисто
- Route groups — публичные страницы в `(marketing)/`, layout изолирован

**Вывод:** ~40% работы сделано (каркас), но 0% работает в продакшне. Главный разрыв — между «код написан» и «код работает с реальными данными на сервере».

## Рекомендация: быстрый старт без VPS

Пока VPS не заказан, можно начать работу через **Docker Compose локально**:

```yaml
# docker-compose.yml (для локальной разработки)
services:
  postgres:
    image: postgres:16
    ports: ["5432:5432"]
    environment:
      POSTGRES_DB: infolog24
      POSTGRES_USER: infolog24
      POSTGRES_PASSWORD: dev_password
  redis:
    image: redis:7-alpine
    ports: ["6379:6379"]
```

Это разблокирует МФ-1 (подключение к реальным данным) без ожидания инфраструктуры.
