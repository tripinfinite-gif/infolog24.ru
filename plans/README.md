## Формат технических планов

- Имя файла: `YYYY-MM-DD-название-функции.md`
- План делится на фазы, у каждой статус `[ ]` (не начато), `[~]` (каркас), `[x]` (готово)
- В конце плана — итоговый блок: реализован целиком или нет, что осталось

## Правила для агентов

- Перед началом работы — проверь, есть ли уже план для этой функции
- После каждой сессии — актуализируй план + запись в журнал MASTER-PLAN-v2 § 8
- Не создавай дублирующих планов — работай с существующим
- Для маркетинговых задач — сначала читай `customer-dna.md`
