# plans/ — технические планы реализации

Здесь хранятся планы реализации функций проекта. Один план = одна функция.

## Формат

- Имя файла: `YYYY-MM-DD-название-функции.md`
- План делится на фазы, у каждой статус `[ ]` (в работе) или `[x]` (готово)
- В конце плана — итоговый блок: реализован целиком или нет, что осталось

## Правила для агентов

- Перед началом работы — проверь, есть ли уже план для этой функции
- После каждой сессии — актуализируй план: отметь сделанные фазы, допиши что изменилось
- Не создавай дублирующих планов — работай с существующим
