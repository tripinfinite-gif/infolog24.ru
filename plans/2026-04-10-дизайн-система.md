## Контекст и ограничения

**Специфика компании:**
- Дептранс негативно относится к посредникам → фирменный стиль должен быть **профессиональным, но не агрессивно-брендовым**
- Целевая аудитория: перевозчики и малый бизнес (часто смотрят с телефона)
- Ключевые ценности: скорость, надёжность, простота
- Рынок с высоким уровнем мошенничества → дизайн должен вызывать **доверие**

**Результат анализа рынка:**
- Проанализированы: DHL, FedEx, Samsara, project44, Uber Freight, Motive, ПЭК, СберЛогистика, Делимобиль
- Изучены тренды: Dribbble, Behance, Awwwards (логистические дашборды и сайты)
- Источники: Smashing Magazine, UX Collective, 99designs, Awwwards, Creative Bloq
- Российская специфика: МойСклад, Контур, Битрикс24

## 1. Цветовая палитра

### Философия выбора

Синий + оранжевый — проверенная B2B-комбинация, подтверждённая лидерами отрасли (Samsara, project44, FedEx). **Синий** = «нам можно доверять, мы работаем 10 лет». **Оранжевый** = «мы делаем быстро, нажми кнопку прямо сейчас». Это прямое визуальное отражение ценностей компании: надёжность + скорость.

## 2. Типографика

### Выбор шрифтов

**Основной шрифт: Inter** — безусловный лидер для SaaS-интерфейсов.
- 414 млрд загрузок на Google Fonts за год (+57% YoY)
- Отличная кириллица, оптимизирован для экранов
- Variable font — один файл, полный контроль над weight
- Используется: Linear, Vercel, Notion, GitHub

**Альтернативный шрифт для заголовков: Manrope** — геометрический, современный.
- Хорошая кириллица, выделяется на фоне Inter
- Добавляет визуальную иерархию без перегрузки

**Запасной вариант (если нужен «русский» характер): Golos Text** — разработка Paratype, бесплатный.

```

### Вертикальный ритм

- Spacing scale: 4px base (Tailwind по умолчанию)
- Между секциями: 80-120px (py-20 / py-24 / py-28)
- Между блоками внутри секции: 32-48px (gap-8 / gap-12)
- Между элементами: 16-24px (gap-4 / gap-6)
- Внутренний padding карточек: 24px (p-6)

### Принцип «78% с мобайла»

78% запросов в логистике приходят с мобильных. Поэтому:
- Все компоненты проектируются **mobile-first**
- Touch targets ≥ 44x44px
- Формы — максимум 2-3 поля на экране
- Таблицы на мобайле → карточный layout
- Фиксированная кнопка «Позвонить» / «Заявка» внизу экрана

### 5.4 Модальные окна

- Overlay: bg-black/50, backdrop-blur-sm
- Modal: bg-white rounded-2xl shadow-2xl, max-w-lg, p-6
- Закрытие: крестик (X) в правом верхнем углу + клик по overlay + Escape
- Анимация: fade-in + scale (Framer Motion, 200ms)
- На мобайле: sheet (выезжает снизу, full-width)

### 5.5 Таблицы

- Для страницы «Цены» и будущего ЛК
- Фиксированная шапка (sticky header)
- Alternating row colors: `odd:bg-neutral-50`
- Hover: `hover:bg-primary-50`
- На мобайле < 768px: горизонтальный скролл или карточный layout
- Сортировка по столбцам (если данных много)
- Фильтры: через выпадающие selects над таблицей

### 5.8 Уведомления / Toast

- Позиция: bottom-right (десктоп), top-center (мобайл)
- 4 типа: success (зелёный), error (красный), warning (жёлтый), info (синий)
- Автоскрытие: 5 секунд
- Иконка + текст + крестик закрытия
- shadcn/ui Sonner — готовый компонент

### 5.9 FAQ / Аккордеон

- Chevron справа, поворачивается при раскрытии (rotate-180, transition 200ms)
- Разделители между пунктами: border-b border-neutral-200
- Текст ответа: text-neutral-600, py-4
- Анимация раскрытия: Radix Accordion (плавная)

## 7. Тёмная / светлая тема

### Решение: только светлая тема для маркетингового сайта

**Обоснование:**
- Целевая аудитория (перевозчики, малый бизнес) привыкла к светлым сайтам
- Тёмная тема не даёт конверсионного преимущества для B2B-лендинга
- Экономит время разработки на старте

**Когда добавить тёмную тему:**
- В будущем ЛК / дашборде — там менеджеры проводят часы и оценят снижение нагрузки на глаза
- Реализация: CSS custom properties через shadcn/ui (переключение встроено)

## 8. Анимации и микровзаимодействия

### Библиотека: Framer Motion (умеренно)

| Элемент | Анимация | Длительность |
|---|---|---|
| Появление секций | fade-in + slide-up (20px) при скролле | 400ms |
| Кнопки | scale(0.98) при нажатии | 100ms |
| Модалки | fade-in + scale(0.95 → 1) | 200ms |
| Карточки | shadow transition при hover | 150ms |
| Счётчики | animated count-up при появлении | 1500ms |
| Навигация mobile | slide-in справа | 250ms |
| Sheet (мобайл) | slide-up снизу | 200ms |
| Skeleton → Content | fade transition | 300ms |

**Принцип:** Анимации должны быть быстрыми и функциональными. Никаких «wow-эффектов» ради эффектов. Целевая аудитория ценит скорость интерфейса, а не спецэффекты.

Поддержка `prefers-reduced-motion`: отключать все анимации.

## 9. Технический стек дизайн-системы

| Слой | Инструмент | Почему |
|---|---|---|
| Компоненты | **shadcn/ui** (Radix + Tailwind) | Полный контроль, нет vendor lock-in, отличная доступность |
| Стили | **Tailwind CSS v4** | CSS-first config, быстрая разработка, mobile-first |
| Анимации | **Framer Motion** | Декларативный API, React-native интеграция |
| Иконки | **Lucide React** | 1500+ иконок, единый стиль, tree-shakeable |
| Шрифты | **Inter + Manrope** (Google Fonts / next/font) | Variable fonts, отличная кириллица |
| Графики (ЛК) | **Recharts** | Простой API, React-компоненты |
| Документация | **Storybook** (опционально, фаза 2) | Каталог компонентов для команды |

### Структура файлов

```
src/

│   └── utils.ts            # cn() и прочие утилиты
