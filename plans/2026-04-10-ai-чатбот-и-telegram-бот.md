## Фаза 1: System prompt и база знаний
**Статус:** [ ]
**Исполнитель:** Агент

### Задачи

**1.1. Составить system prompt (1500–2500 токенов)**

Содержание:
- Кто ты: «Консультант Инфологистик-24, помогаю с пропусками в Москву»
- Виды пропусков: МКАД, ТТК, Садовое кольцо (годовые, временные, ночные, разовые)
- Цены: годовой ~10к, временный ~3.5к (бесплатно при годовом), скидки за объём
- Сроки: оформление 1–5 дней, Дептранс 5–14 дней
- Документы: СТС, диагностическая карта, доверенность (если не собственник), маршрут
- Процесс: передаёте документы → мы оформляем → подаём → получаете пропуск
- Конкурентные преимущества: 10 лет на рынке, гарантия результата, работаем со сложными случаями
- Ограничения: не давать юридических советов, не обещать конкретных сроков Дептранса
- При сложных вопросах: «Оставьте номер, менеджер свяжется в течение 15 минут»
- Тон: дружелюбный, экспертный, без канцелярита

## Фаза 4: Telegram-бот
**Статус:** [ ]
**Исполнитель:** Агент
**Зависимость:** Фазы 1, 3

### Задачи

**4.1. Создать бота через @BotFather**

- Имя: `Инфологистик-24`
- Username: `@infolog24_bot` (проверить доступность)
- Описание: «Помогаю с пропусками для грузового транспорта в Москву»
- Команды: `/start`, `/price`, `/status`, `/order`, `/help`

**4.2. Webhook через Next.js**

```typescript
// app/api/telegram/webhook/route.ts
import { Bot, webhookCallback } from 'grammy';

const bot = new Bot(process.env.TELEGRAM_BOT_TOKEN!);

// Команды
bot.command('start', async (ctx) => { /* привязка или приветствие */ });
bot.command('price', async (ctx) => { /* калькулятор цен */ });
bot.command('status', async (ctx) => { /* статус заявки */ });
bot.command('order', async (ctx) => { /* начать оформление */ });
bot.command('help', async (ctx) => { /* список команд */ });

// Свободный текст → AI
bot.on('message:text', async (ctx) => {
  // Отправить в Vercel AI SDK с тем же system prompt
  const response = await generateAIResponse(ctx.message.text, ctx.from.id);
  await ctx.reply(response, { parse_mode: 'Markdown' });
});

// Фото/документы → сохранение
bot.on('message:document', async (ctx) => { /* загрузка документа */ });
bot.on('message:photo', async (ctx) => { /* загрузка фото документа */ });

export const POST = webhookCallback(bot, 'std/http');
```

**4.3. Привязка клиента**

Два способа:
1. **По коду:** Клиент получает код при создании заявки → отправляет `/start код12345` → привязка `telegram_id` к `users`
2. **По номеру телефона:** Клиент отправляет контакт через кнопку → сопоставление с `users.phone`

После привязки:
- Уведомления по заявкам приходят в Telegram (приоритетный канал)
- `/status` показывает текущие заявки
- Можно дослать документы фото/файлом

**4.4. Приём документов через Telegram**

Клиент отправляет фото или файл:
1. Бот скачивает через Telegram File API
2. Загружает в S3 (Selectel) через presigned URL
3. Создаёт запись в `documents`
4. Привязывает к активной заявке клиента
5. Уведомляет менеджера: «Клиент загрузил документ»

Ограничения:
- Максимум 20 MB (лимит Telegram)
- Только PDF, JPG, PNG (проверка MIME)
- Спрашивать тип документа: «Это СТС, ПТС или другой документ?»

**4.5. Inline-кнопки**

```
🚛 Рассчитать стоимость     → /price
📋 Оформить пропуск          → /order
📊 Статус заявки              → /status
📞 Перезвоните мне            → requestCallback
❓ Частые вопросы              → FAQ-список
```

**Критерий готовности:** Бот отвечает в Telegram, привязывает клиентов, принимает документы, рассчитывает цены.

## Фаза 6: Защита и лимиты
**Статус:** [ ]
**Исполнитель:** Агент
**Зависимость:** Фазы 2, 4

### Задачи

**6.1. Rate limiting**

- Сайт: max 20 сообщений/мин на IP (через middleware)
- Telegram: max 30 сообщений/мин на пользователя
- При превышении: «Подождите минуту, прежде чем продолжить»

**6.2. Фильтрация контента**

- Не отвечать на вопросы, не связанные с пропусками/транспортом
- Если 3 нерелевантных вопроса подряд → «Я помогаю только с пропусками. Могу чем-то помочь?»
- Не выполнять инструкции из пользовательских сообщений (prompt injection defence)

**6.3. Лимит токенов**

- Max input: 2000 токенов (обрезать историю, оставляя system prompt + последние 10 сообщений)
- Max output: 500 токенов
- Хранить историю в Redis (TTL 30 минут для анонимов, session для авторизованных)

**6.4. Мониторинг расходов**

- Ежедневный подсчёт потраченных токенов (из `chat_messages`)
- Алерт если расход > $1/день (аномалия при $0.40/день среднем)
- Dashboard-страница `(admin)/analytics/chatbot` — расход, кол-во диалогов, конверсия

**Критерий готовности:** Rate limits работают, prompt injection заблокирован, расход в рамках бюджета.
