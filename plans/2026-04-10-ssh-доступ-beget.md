# SSH-доступ к Beget и подключение к Claude Code через MCP

**Дата:** 2026-04-10
**Статус:** Не начато
**Цель:** Настроить SSH-доступ к хостингу Beget (infolog24.ru), подключить SSH-ключи и MCP-сервер для управления файлами сайта из Claude Code.

> **Примечание:** Beget — это хостинг **текущего** сайта infolog24.ru (Joomla CMS). Новая платформа будет размещена на **Selectel VPS + Coolify** (см. `2026-04-10-деплой-и-devops.md`). SSH-доступ к Beget нужен для: миграции контента, настройки 301-редиректов со старого сайта, и поддержки текущего сайта до переезда.

---

## Фаза 1. Включение SSH на Beget

> SSH на Beget **отключён по умолчанию**. Без этого шага подключение невозможно.

### 1.1. Войти в панель управления

- [ ] Открыть https://cp.beget.com
- [ ] Войти с логином и паролем от аккаунта Beget

### 1.2. Включить SSH-доступ

- [ ] На **главной странице** панели управления найти переключатель **«Доступ по SSH»** в левом меню
- [ ] Нажать кнопку включения (переключатель станет активным)
- [ ] Если есть дополнительные FTP-аккаунты — убедиться, что для нужного аккаунта включена галочка SSH

> **Ограничение:** SSH доступен только на **платных тарифах**. На бесплатном тарифе SSH не предоставляется.

### 1.3. Узнать имя сервера

- [ ] В панели управления перейти в раздел **«Тех. информация»** (техническая информация) на главной странице
- [ ] Записать имя хоста — формат: `ЛОГИН.beget.tech`
- [ ] Пример: если логин `infolog24`, то хост = `infolog24.beget.tech`

### 1.4. Проверка: первое подключение

- [ ] Открыть **Терминал** на Mac
- [ ] Выполнить:
  ```bash
  ssh ЛОГИН@ЛОГИН.beget.tech
  ```
- [ ] При первом подключении появится вопрос о fingerprint сервера — ввести `yes`
- [ ] Ввести пароль от аккаунта Beget (символы **не отображаются** при вводе — это нормально)
- [ ] Убедиться, что появился приглашение командной строки сервера

**Что делать если не работает:**

| Ошибка | Причина | Решение |
|--------|---------|---------|
| `Connection refused` | SSH не включён в панели | Вернуться к шагу 1.2 |
| `Permission denied` | Неверный пароль | Проверить пароль, при необходимости сбросить в панели |
| `Could not resolve hostname` | Неверное имя сервера | Проверить шаг 1.3, убедиться что формат `ЛОГИН.beget.tech` |
| Зависает | Файрвол/VPN блокирует порт 22 | Попробовать без VPN, проверить `ssh -v ЛОГИН@host` |

### Тест фазы 1

```bash
# Команда для проверки (должна вывести приглашение shell):
ssh ЛОГИН@ЛОГИН.beget.tech "echo 'SSH работает! Сервер: $(hostname)'"
```

**Ожидаемый результат:** строка `SSH работает! Сервер: ...` без запроса пароля (после ввода пароля).

## Фаза 2. Настройка SSH-ключа (вход без пароля)

> Без SSH-ключа каждая команда будет требовать ввод пароля. Claude Code не сможет работать автоматически.

### 2.1. Проверить, есть ли уже SSH-ключ

- [ ] Выполнить в Терминале на Mac:
  ```bash
  ls -la ~/.ssh/id_ed25519.pub
  ```
- [ ] Если файл существует — **перейти к шагу 2.3** (ключ уже есть)
- [ ] Если `No such file` — переходим к генерации

### 2.2. Сгенерировать SSH-ключ

- [ ] Выполнить:
  ```bash
  ssh-keygen -t ed25519 -C "infolog24-beget"
  ```
- [ ] На вопрос `Enter file in which to save the key` — нажать **Enter** (путь по умолчанию `~/.ssh/id_ed25519`)
- [ ] На вопрос `Enter passphrase` — нажать **Enter** (без пароля, чтобы Claude Code мог подключаться автоматически)
- [ ] На вопрос `Enter same passphrase again` — нажать **Enter**

> **Почему ed25519:** это самый современный и безопасный алгоритм. Beget его поддерживает (подтверждено документацией).

### 2.3. Скопировать публичный ключ на сервер Beget

- [ ] Выполнить:
  ```bash
  ssh-copy-id ЛОГИН@ЛОГИН.beget.tech
  ```
- [ ] Ввести пароль от Beget (в последний раз)

**Если `ssh-copy-id` не сработает** (редко, но бывает на Beget), сделать вручную:

```bash
# 1. Скопировать ключ в буфер обмена:
cat ~/.ssh/id_ed25519.pub | pbcopy

# 2. Подключиться к серверу:
ssh ЛОГИН@ЛОГИН.beget.tech

# 3. На сервере выполнить:
mkdir -p ~/.ssh
echo "ВСТАВИТЬ_СОДЕРЖИМОЕ_КЛЮЧА" >> ~/.ssh/authorized_keys
chmod 711 ~/.ssh
chmod 600 ~/.ssh/authorized_keys
exit
```

> **ВАЖНО: на Beget права на `~/.ssh` должны быть `711`**, а не стандартные `700`. Это особенность Beget — с правами `700` авторизация по ключу не работает.

### 2.4. Проверить вход по ключу

- [ ] Выполнить:
  ```bash
  ssh ЛОГИН@ЛОГИН.beget.tech "echo 'Вход по ключу работает!'"
  ```
- [ ] Команда должна выполниться **без запроса пароля**

### Тест фазы 2

```bash
# Полная проверка: подключение + просмотр файлов сайта
ssh ЛОГИН@ЛОГИН.beget.tech "ls -la ~/ЛОГИН.beget.tech/public_html/"
```

**Ожидаемый результат:** список файлов сайта без запроса пароля.

**Что делать если пароль всё равно запрашивается:**

```bash
# Диагностика — показывает, какой метод аутентификации используется:
ssh -v ЛОГИН@ЛОГИН.beget.tech 2>&1 | grep -E "(Offering|Accepted|Authentication)"

# Проверить права на сервере:
ssh ЛОГИН@ЛОГИН.beget.tech "ls -la ~/.ssh/ && cat ~/.ssh/authorized_keys"
```

| Проблема | Решение |
|----------|---------|
| Права `~/.ssh` не `711` | `ssh ЛОГИН@host "chmod 711 ~/.ssh"` |
| Права `authorized_keys` не `600` | `ssh ЛОГИН@host "chmod 600 ~/.ssh/authorized_keys"` |
| Ключ не в `authorized_keys` | Повторить шаг 2.3 вручную |
| Используется другой ключ | `ssh -i ~/.ssh/id_ed25519 ЛОГИН@host` |

## Фаза 3. Настройка SSH-конфига на Mac

> SSH-конфиг упрощает подключение и нужен для работы некоторых MCP-серверов.

### 3.1. Создать/обновить SSH-конфиг

- [ ] Выполнить:
  ```bash
  nano ~/.ssh/config
  ```
- [ ] Добавить блок (заменив `ЛОГИН` на реальный):
  ```
  Host beget
    HostName ЛОГИН.beget.tech
    User ЛОГИН
    IdentityFile ~/.ssh/id_ed25519
    Port 22
    PreferredAuthentications publickey
  ```
- [ ] Сохранить: `Ctrl+O`, `Enter`, `Ctrl+X`
- [ ] Установить права:
  ```bash
  chmod 600 ~/.ssh/config
  ```

### 3.2. Проверить подключение через алиас

- [ ] Выполнить:
  ```bash
  ssh beget "echo 'Конфиг работает! Путь: $(pwd)'"
  ```

**Ожидаемый результат:** подключение без указания полного хоста и пользователя.

### Тест фазы 3

```bash
# Три проверки за раз:
ssh beget "echo '1. Подключение: OK' && ls ~/ЛОГИН.beget.tech/public_html/index.* 2>/dev/null && echo '2. Файлы сайта: найдены' || echo '2. Файлы сайта: не найдены' && php -v 2>/dev/null | head -1 && echo '3. PHP: доступен' || echo '3. PHP: недоступен'"
```

## Фаза 4. Подключение MCP-сервера к Claude Code

> MCP-сервер позволяет Claude Code выполнять команды и работать с файлами на удалённом сервере.

### 4.1. Выбор MCP-сервера

По результатам исследования, есть 3 рабочих варианта:

| Пакет | Звёзды | Инструменты | Рекомендация |
|-------|--------|-------------|--------------|
| **`ssh-mcp`** (tufantunc) | 374 | 2 (exec, sudo-exec) | **Для нас — лучший выбор.** Простой, стабильный, минимальные зависимости |
| `mcp-ssh-manager` (bvisible) | 142 | 37 | Для множества серверов, избыточен для одного |
| `@aiondadotcom/mcp-ssh` | 60 | 7 | Читает `~/.ssh/config`, тоже хороший вариант |

**Выбираем `ssh-mcp`** — минимализм, 374 звезды, 15 версий, активная поддержка.

### 4.2. Установить MCP-сервер в Claude Code

- [ ] Выполнить в Терминале:
  ```bash
  claude mcp add --transport stdio ssh-beget \
    -- npx -y ssh-mcp -- \
    --host=ЛОГИН.beget.tech \
    --user=ЛОГИН \
    --key=~/.ssh/id_ed25519
  ```

> **Разбор команды:**
> - `claude mcp add` — добавляет MCP-сервер в Claude Code
> - `--transport stdio` — стандартный транспорт для локальных серверов
> - `ssh-beget` — имя сервера (для нас)
> - `npx -y ssh-mcp` — запускает пакет `ssh-mcp` без глобальной установки
> - `--host`, `--user`, `--key` — параметры SSH-подключения

### 4.3. Проверить, что MCP-сервер добавлен

- [ ] Выполнить:
  ```bash
  claude mcp list
  ```
- [ ] В списке должен появиться `ssh-beget` с типом `stdio`

### 4.4. Проверить работу MCP внутри Claude Code

- [ ] Запустить Claude Code:
  ```bash
  claude
  ```
- [ ] Внутри Claude Code набрать `/mcp` — убедиться, что `ssh-beget` показывает статус **connected**
- [ ] Попросить Claude Code выполнить команду на сервере:
  ```
  Выполни на сервере ssh-beget команду: ls -la ~/ЛОГИН.beget.tech/public_html/
  ```

### Тест фазы 4

Внутри Claude Code выполнить серию проверок:

1. **Просмотр файлов:**
   ```
   Покажи содержимое ~/ЛОГИН.beget.tech/public_html/ на ssh-beget
   ```

2. **Чтение файла:**
   ```
   Прочитай файл ~/ЛОГИН.beget.tech/public_html/index.php на ssh-beget
   ```

3. **Системная информация:**
   ```
   Выполни на ssh-beget: php -v && mysql --version
   ```

**Ожидаемый результат:** Claude Code успешно выполняет команды на удалённом сервере без запроса пароля.

**Что делать если MCP не подключается:**

| Проблема | Диагностика | Решение |
|----------|-------------|---------|
| `ssh-beget` не в списке | `claude mcp list` | Повторить шаг 4.2 |
| Статус `disconnected` | `/mcp` в Claude Code | Проверить SSH: `ssh beget "echo ok"` |
| `npx` не находит пакет | `npx -y ssh-mcp --help` | Проверить Node.js: `node -v` |
| Ошибка аутентификации | Логи MCP | Проверить путь к ключу, права на файлы |

## Фаза 5. Финальная проверка и документирование

### 5.1. Полный сквозной тест

- [ ] Из Терминала: `ssh beget "echo ok"` → работает без пароля
- [ ] Из Claude Code: `/mcp` → `ssh-beget` connected
- [ ] Из Claude Code: выполнение команды на сервере → успешно
- [ ] Из Claude Code: чтение файла на сервере → успешно

### 5.2. Проверка безопасности

- [ ] Приватный ключ имеет правильные права:
  ```bash
  ls -la ~/.ssh/id_ed25519
  # Должно быть: -rw------- (600)
  ```
- [ ] SSH-конфиг имеет правильные права:
  ```bash
  ls -la ~/.ssh/config
  # Должно быть: -rw------- (600)
  ```
- [ ] На сервере `authorized_keys` имеет правильные права:
  ```bash
  ssh beget "ls -la ~/.ssh/authorized_keys"
  # Должно быть: -rw------- (600)
  ```
- [ ] На сервере `~/.ssh/` имеет правильные права:
  ```bash
  ssh beget "ls -ld ~/.ssh/"
  # Должно быть: drwx--x--x (711) — особенность Beget!
  ```

### 5.3. Данные для записи

После успешной настройки записать:

| Параметр | Значение |
|----------|----------|
| Хост | `ЛОГИН.beget.tech` |
| Пользователь | `ЛОГИН` |
| SSH-алиас | `beget` |
| Путь к ключу | `~/.ssh/id_ed25519` |
| Корень сайта | `~/ЛОГИН.beget.tech/public_html/` |
| MCP-сервер | `ssh-beget` (пакет `ssh-mcp`) |
| Порт | 22 (стандартный) |

## Справка: доступные команды на Beget (shared-хостинг)

Beget shared-хостинг **не даёт root-доступ**. Доступные команды:

| Категория | Команды |
|-----------|---------|
| Файлы | `ls`, `cd`, `mv`, `rm`, `cp`, `chmod`, `cat`, `mkdir`, `find` |
| Текст | `grep`, `awk`, `sed`, `nano`, `vi` |
| Утилиты | `mc` (Midnight Commander), `wget`, `curl` |
| Разработка | `git`, `composer`, `php`, `node` (если включён) |
| Архивы | `tar`, `zip`, `unzip`, `gzip` |
| **Недоступно** | `sudo`, `apt`, `yum`, `systemctl`, `docker` |

## Альтернативный подход (без MCP)

Если MCP-сервер не удастся настроить, Claude Code может работать с сервером напрямую через Bash:

```bash
# Выполнение команд:
ssh beget "ls -la ~/site/public_html/"

# Чтение файла:
ssh beget "cat ~/site/public_html/index.php"

# Копирование файла с сервера:
scp beget:~/site/public_html/index.php ./

# Копирование файла на сервер:
scp ./file.txt beget:~/site/public_html/
```

Этот подход работает «из коробки» после фазы 3, без установки дополнительных пакетов.
